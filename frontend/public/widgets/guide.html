Weekly Event Panel — Hướng dẫn tích hợp

Chèn panel đếm ngược "Sự kiện tuần" vào trang của bạn bằng một thẻ script. Tài liệu này dành cho tích hợp từ bên ngoài.

1. Panel này là gì?

Một file JavaScript dựng sẵn. Bạn chỉ cần:

  • Chèn thẻ <script> trỏ tới file weekly-event-panel.js.
  • Đặt một thẻ <div> làm chỗ gắn — script tự dựng panel vào đó.

Panel sẽ tự gọi API để biết sự kiện tuần có đang mở đăng ký không, rồi hiện đồng hồ đếm ngược. Giao diện panel nằm trong Shadow DOM nên không ảnh hưởng CSS trang của bạn và ngược lại.

2. Cách nhúng

Cách 1 — Tự khởi tạo (đơn giản nhất)

Thêm thuộc tính data-auto. Script tự tìm thẻ div theo data-target và khởi tạo.

<div id="we-panel"></div>

<script
  src="https://YOUR_DOMAIN/widgets/weekly-event-panel.js"
  data-auto
  data-token="<JWT của học sinh>"
  data-api-base="https://YOUR_API/api"
  data-weekly-event-url="https://YOUR_APP/weekly-event"
  data-target="#we-panel"></script>

Cách 2 — Tự gọi hàm khởi tạo (linh hoạt hơn)

Không dùng data-auto. Sau khi script tải xong, gọi WeeklyEventPanel.init() khi bạn muốn.

<div id="we-panel"></div>
<script src="https://YOUR_DOMAIN/widgets/weekly-event-panel.js"></script>
<script>
  var panel = WeeklyEventPanel.init('#we-panel', {
    token: '<JWT>',
    apiBase: 'https://YOUR_API/api',
    container: '#game-stage',   // neo panel trong khung này; bỏ trống = phủ toàn màn hình
    onJoin: function (ctx) {     // bấm nút vào sự kiện — bạn tự quyết định điều hướng
      window.location.href = ctx.weeklyEventUrl;
    }
  });

  // Khi rời màn hình, gỡ panel:
  // panel.destroy();
</script>
Token lấy ở đâu? Là JWT của học sinh mà hệ thống của bạn cấp (cùng token dùng để gọi API game). Nếu không truyền token, panel vẫn hiện được nhưng không biết học sinh đã tham gia hay chưa.

3. Các tuỳ chọn (options)

TênMặc địnhCông dụng
token—JWT học sinh để gọi API. Có thể bỏ trống.
apiBasedomain hiện tại + /apiĐịa chỉ gốc của API.
weeklyEventUrldomain hiện tại + /weekly-eventTrang sẽ mở khi bấm "Tham gia ngay".
container—Chọn khung để đặt panel bên trong (vd '#game-stage'). Bỏ trống thì panel nổi phủ toàn màn hình.
designWidth640Bề rộng chuẩn của panel (px). Nếu chỗ hẹp hơn, panel tự thu nhỏ đều.
minScale0.5Mức thu nhỏ tối đa (0.5 = nhỏ nhất còn 50%).
pollInterval0Số mili-giây tự kiểm tra lại API. Để 0 là tắt.
onJoin—Hàm chạy khi bấm nút vào sự kiện (dùng chung cho cả "Tham gia ngay" và "Quay lại phòng chờ"). Nếu có hàm này, panel không tự điều hướng — bạn tự làm.

Dữ liệu truyền vào onJoin(ctx)

TrườngÝ nghĩa

4. Khi nào panel hiện / ẩn?

  • Đang mở đăng ký → panel hiện, đếm ngược tới lúc hết hạn tham gia. Hết giờ thì tự ẩn.
  • Chưa tới giờ mở → panel ẩn, tự canh đúng giờ mở rồi mới hiện.
  • Đã vào thi / đã đóng → panel ẩn.
  • Màu đồng hồ: còn > 60 giây xanh, ≤ 60 giây cam, ≤ 20 giây đỏ.
  • Học sinh có thể Thu gọn panel thành huy hiệu 🏆 nhỏ ở mép trái (kéo lên xuống được), hoặc Tắt (ẩn trong phiên đó).

5. Thử nghiệm trực tiếp

Bước 1. Nhập domain nơi bạn host file script, rồi bấm Tải script. Bước 2. Chỉnh các tuỳ chọn để xem panel đổi theo (dùng dữ liệu giả, không gọi API thật).

Sẽ tải: https://your-domain.com/widgets/weekly-event-panel.js
Chưa tải script.
Kéo để xem đồng hồ đổi màu xanh → cam → đỏ.
Panel ẩn cho tới khi hết số giây này rồi tự hiện.
Bật để đổi nút thành "Quay lại phòng chờ".
"Toàn màn hình" sẽ phủ lên cả trang (đúng cho app nhúng toàn màn).
"Chạy hàm onJoin" sẽ hiện thông tin thay vì rời trang.
Nhập domain và bấm Tải script để bắt đầu.
